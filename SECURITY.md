# Security policy

## Reporting a vulnerability

If you find a security issue in this repository — in a skill, prompt, eval fixture, or anything else — please report it privately rather than opening a public issue:

- **Preferred:** [Report a vulnerability](https://github.com/TeamRetroHQ/teamretro-skills/security/advisories/new) via GitHub's private reporting.
- **Email:** [security@teamretro.com](mailto:security@teamretro.com)

We'll acknowledge your report promptly and keep you informed as we investigate and fix.

## Scope notes

These skills instruct AI agents. Reports we especially want to hear about:

- Skill or prompt-pack text that could cause an agent to leak secrets, credentials, or private data (the skills explicitly forbid this — tell us if you find a gap).
- Instructions that could cause unintended writes or destructive actions, including via the TeamRetro MCP server.
- Anything in the examples or eval fixtures that looks like real customer data (it should all be illustrative).

For security questions about the TeamRetro product itself, see the [TeamRetro Trust Center](https://trust.teamretro.com) or contact [security@teamretro.com](mailto:security@teamretro.com).
